# 네이버 오픈API 공통 가이드

네이버 오픈API를 사용하여 애플리케이션을 개발할 때 필요한 공통적인 사전 준비 사항, 호출 방법, 에러 대응 방법에 대한 개발 명세입니다.

---

## 1. API 구분
네이버 오픈API는 크게 두 가지 방식으로 구분됩니다.

| 구분 | 인증 방식 | 설명 | 대상 API |
| :--- | :--- | :--- | :--- |
| **비로그인 방식 API** | HTTP 헤더에 Client ID/Secret 전송 | 로그인 과정 없이 API 키 인증만으로 사용 가능한 API | 검색 (블로그, 뉴스, 카페글, 쇼핑), 데이터랩 등 |
| **로그인 방식 API** | OAuth 2.0 기반 Access Token 인증 | 네이버 아이디로 로그인하여 사용자 권한을 획득해야 하는 API | 네이버 로그인, 지도, 캘린더 등 |

---

## 2. 사전 준비 사항
API를 호출하기 위해서는 네이버 개발자 센터에서 애플리케이션을 등록하고 인증 정보를 발급받아야 합니다.

### 애플리케이션 등록 단계
1. **네이버 개발자 센터** 접속 및 로그인 (단체 회원의 경우 대표 단체 아이디 권장)
2. **Application > 애플리케이션 등록** 메뉴 선택
3. 약관 동의 및 계정 정보(최초 1회 휴대폰 인증) 등록
4. **애플리케이션 이름** 입력 및 **사용 API** 설정 (예: 검색, 데이터랩 등 필요한 API 추가)
5. **서비스 환경** 설정 (웹 URL 또는 앱 패키지명 입력)
6. 등록 완료 후 **Client ID** 및 **Client Secret** 확인

---

## 3. 공통 API 호출 규격

### HTTP 요청 헤더
비로그인 방식 API를 호출할 때는 항상 다음 두 가지 헤더를 요청에 포함해야 합니다.

| 헤더 이름 | 필수 여부 | 설명 |
| :--- | :---: | :--- |
| `X-Naver-Client-Id` | 필수 (Y) | 애플리케이션 등록 후 발급받은 클라이언트 아이디 값 |
| `X-Naver-Client-Secret` | 필수 (Y) | 애플리케이션 등록 후 발급받은 클라이언트 시크릿 값 |

---

## 4. 공통 오류 코드

네이버 오픈API 호출 실패 시 반환되는 주요 공통 오류 코드 및 형식입니다.

### 오류 메시지 형식 (JSON)
```json
{
  "errorMessage": "오류 설명 메시지",
  "errorCode": "오류 코드"
}
```

### 공통 오류 코드 명세
| HTTP 상태 코드 | 오류 코드 | 설명 / 조치 방안 |
| :---: | :---: | :--- |
| **400** | SE01 | **잘못된 쿼리 요청**: 요청 파라미터가 누락되었거나 값의 형식이 올바르지 않은 경우 |
| **403** | 024 / SE05 | **인증 실패 / 권한 없음**: Client ID/Secret 값이 유효하지 않거나, 애플리케이션의 'API 설정' 탭에서 해당 API 권한을 켜지 않은 경우 |
| **429** | 028 | **호출 한도 초과**: API별 제공되는 일일 호출 한도(Quota)를 모두 사용한 경우 |
| **500** | SE99 | **시스템 내부 에러**: 네이버 서버 내부적인 에러로, 개발자 포럼 등을 통해 문의 필요 |
